Buttons

Nova UI buttons are interactive elements that trigger actions or navigation. They come in various appearances, variants, and sizes to meet different design needs.

Introduction

The NovaButtonComponent is a versatile button component with cosmic-themed styling that can be customized through several properties:

  • 8 Appearances - Choose from basic, raised, stroked, flat, icon, fab, mini-fab, and extended-fab
  • 3 Semantic Variants - Basic, Call-to-Action (CTA), and Link variants for different emphasis levels
  • 3 Sizes - Small, medium, and large sizes to fit your interface needs
  • Animation Levels - None, normal, or extra animation effects for different levels of interactivity
  • States - Normal, disabled, and loading states
  • Icon Support - Left or right icon positioning

Import

import { NovaButtonComponent } from 'nova-ui';

Button Appearances

The appearance property defines the visual style of the button. Nova UI offers eight different appearances to match your design needs.

Basic Basic appearance="basic"
Raised Raised appearance="raised"
Stroked Stroked appearance="stroked"
Flat Flat appearance="flat"
Icon appearance="icon" icon="★"
FAB appearance="fab" icon="+"
Mini FAB appearance="mini-fab" icon="-"
Extended FAB Extended appearance="extended-fab" icon="→"

Button Variants

The variant property defines the semantic meaning of the button. Use these variants to establish visual hierarchy in your interfaces.

Basic Variant Basic variant="basic"
CTA Variant CTA variant="cta"
Link Variant Link variant="link"

Usage Guidance

  • Basic variant - Use for standard actions that don't need to stand out
  • CTA variant - Use for primary actions that should draw attention
  • Link variant - Use for navigation or secondary actions

Button Sizes

The size property adjusts the size of the button. Choose from three size options to fit your interface needs.

Small Small size="small"
Medium (Default) Medium size="medium"
Large Large size="large"

Button States

Buttons can be in normal, disabled, or loading states. Use the disabled and loading properties to control the state.

Normal Normal <nova-button>
Disabled Disabled [disabled]="true"
Loading Loading [loading]="true"

Button with Icons

Add an icon to a button using the icon property. Control the icon position with iconPosition.

Icon Left (Default) Back icon="←" iconPosition="left"
Icon Right Next icon="→" iconPosition="right"

Animation Levels

Control the amount of animation applied to buttons with the animation property.

None No Animation animation="none"
Normal (Default) Normal Animation animation="normal"
Extra Extra Animation animation="extra"

Common Use Cases

Here are some examples of how to combine button properties for common UI patterns.

Use a combination of link, basic, and CTA variants for form actions:

Cancel Save Draft Submit 
<nova-button variant="link">Cancel</nova-button>
  <nova-button>Save Draft</nova-button>
  <nova-button variant="cta">Submit</nova-button>

Use link and CTA variants for dialog actions:

Cancel Confirm 
<nova-button variant="link">Cancel</nova-button>
  <nova-button variant="cta">Confirm</nova-button>

Use icon buttons for compact controls:

<nova-button appearance="icon" icon="←"></nova-button>
  <nova-button appearance="icon" icon="↑"></nova-button>
  <nova-button appearance="icon" icon="↓"></nova-button>
  <nova-button appearance="icon" icon="→"></nova-button>

Accessibility

Nova UI buttons are designed with accessibility in mind. Here are some key accessibility features:

  • Uses native <button> element for proper keyboard navigation and screen reader support
  • Maintains focus styles for keyboard navigation
  • Sets proper ARIA attributes for disabled and loading states
  • Provides sufficient color contrast between button text and background
  • Supports custom icons with appropriate labeling

Tip

Always provide a clear, descriptive label for buttons. For icon-only buttons, consider adding an aria-label attribute to the parent element that contains the button for better screen reader support.
